== 概要

MoneyBook は収支記録・月次サマリー・チャート表示・CSV エクスポートを提供するシングルページ家計簿アプリである。
バニラ JS（ES Modules）で実装し、LocalStorage にデータを永続化する。

== モジュール一覧

[cols="2,1,3"]
|===
| ファイル | クラス / 関数 | 役割

| `src/core/kakeibo-engine.js` | `KakeiboEngine` | アプリ全体の初期化・イベント委譲・画面更新
| `src/core/budget-service.js` | `BudgetService` | 収支レコードの CRUD・集計ロジック
| `src/managers/storage-manager.js` | `StorageManager` | LocalStorage への読み書き
| `src/managers/state-manager.js` | `StateManager` | 表示月・モーダル状態の保持
| `src/managers/chart-manager.js` | `ChartManager` | Chart.js ドーナツチャートのラッパー
| `src/managers/csv-manager.js` | `CsvManager` | CSV 生成とダウンロード
| `src/ui/header.js` | `renderHeader()` | ヘッダー HTML テンプレート
| `src/ui/footer.js` | `renderFooter()` | フッター HTML テンプレート
| `src/ui/components.js` | `renderRecordItem()` / `renderCategoryOptions()` | 記録カード・カテゴリ選択肢 HTML テンプレート
| `src/utils/validator.js` | `validateRecord()` / `validateBudget()` | バリデーション
| `src/utils/xss.js` | `escapeHtml()` | XSS フィルタリング
| `src/utils/formatter.js` | `formatCurrency()` 他 | 通貨・日付フォーマット
|===

== KakeiboEngine

アプリ全体の初期化・イベント委譲・画面更新を統括するメインクラス。

=== コンストラクタ

`new KakeiboEngine()` で `StorageManager`・`StateManager`・`BudgetService`・`ChartManager`・`CsvManager` を生成する。

=== メソッド一覧

[cols="2,1,3"]
|===
| メソッド | 戻り値 | 説明

| `init()` | void | ヘッダー・フッターを描画し、イベントをバインドし、初回データを描画する
| `_renderShell()` | void | ヘッダー（ロゴ＋月ナビ）とフッターを DOM に挿入する
| `_bindEvents()` | void | document にクリック・change・submit イベントをまとめて委譲する
| `_prevMonth()` | void | 表示月を前月に移動し、ヘッダーと全コンテンツを更新する
| `_nextMonth()` | void | 表示月を翌月に移動し、ヘッダーと全コンテンツを更新する
| `_refresh()` | void | サマリー・記録一覧・チャートをまとめて再描画する
| `_renderSummary(summary)` | void | 収入・支出・残高カードのテキストを更新する
| `_renderRecords(records)` | void | 記録一覧を描画する（0 件なら空メッセージを表示）
| `_renderChart(categoryData)` | void | 支出カテゴリのドーナツチャートを描画する（0 件なら空メッセージ）
| `_updateHeader()` | void | ヘッダー HTML のみを再生成して月表示を更新する
| `_openAddModal()` | void | フォームを初期化して追加モーダルを表示する
| `_openEditModal(id)` | void | 指定 ID のレコード値をフォームに設定して編集モーダルを表示する
| `_closeModal()` | void | モーダルを非表示にしてモーダル状態をリセットする
| `_updateCategoryOptions(type, selected)` | void | 種別に応じたカテゴリ選択肢を `<select>` に反映する
| `_submitRecord()` | void | フォーム値を取得して追加または更新を実行する（エラー時はメッセージ表示）
| `_deleteRecord()` | void | confirm ダイアログ確認後に編集中レコードを削除する
| `_exportCsv()` | void | 現在月のレコードを CSV ファイルとしてダウンロードする
|===

== BudgetService

収支レコードの CRUD と集計を担当する。`StorageManager` を DI で受け取るためユニットテストが可能。

=== コンストラクタ

[source,javascript]
----
new BudgetService(storage: StorageManager)
----

=== メソッド一覧

[cols="2,1,3"]
|===
| メソッド | 戻り値 | 説明

| `addRecord(data)` | string (id) | バリデーション後にレコードを追加し、生成した ID を返す。エラー時は Error をスロー
| `updateRecord(id, data)` | void | バリデーション後に指定 ID のレコードを更新する。ID 不在時は Error をスロー
| `deleteRecord(id)` | void | 指定 ID のレコードを削除する
| `getRecordsByMonth(yearMonth)` | Record[] | 指定月のレコードを日付降順で返す（`yearMonth` は `YYYY-MM` 形式）
| `getSummary(yearMonth)` | `{income, expense, balance}` | 指定月の収入・支出・残高の集計オブジェクトを返す
| `getExpenseByCategory(yearMonth)` | `{[category]: number}` | 指定月の支出をカテゴリ別に集計したオブジェクトを返す
|===

=== Record オブジェクト構造

[source,json]
----
{
  "id":        "1712345678901-abc123",
  "date":      "2026-04-01",
  "type":      "expense",
  "category":  "食費",
  "amount":    1500,
  "memo":      "スーパー",
  "createdAt": "2026-04-01T10:00:00.000Z"
}
----

== StorageManager

[cols="2,1,3"]
|===
| メソッド | 戻り値 | 説明

|===

== validator.js

=== 関数一覧

[cols="2,1,3"]
|===
| 関数 | 戻り値 | 説明

| `validateRecord({date, type, category, amount})` | string[] | レコード入力値を検証し、エラーコードの配列を返す（正常時は空配列）
| `validateBudget({category, limit})` | string[] | 予算設定入力値を検証し、エラーコードの配列を返す（正常時は空配列）
|===

=== validateRecord 検証ルール

[cols="2,4"]
|===
| フィールド | ルール

| `date` | 必須・`/^\d{4}-\d{2}-\d{2}$/` にマッチすること
| `type` | `RECORD_TYPES` の値（`income` / `expense`）であること
| `category` | `INCOME_CATEGORIES` または `EXPENSE_CATEGORIES` に含まれること
| `amount` | 必須・`Number()` で数値化できる・1 以上の整数であること
|===
