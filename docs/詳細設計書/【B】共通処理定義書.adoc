== 概要

本書は MONEYBOOK（家計簿アプリ kakeibo-app）全体で使用する共通ユーティリティの仕様を記載します。
XSS対策・入力バリデーション・ストレージ管理を対象とします。

---

== 1. escapeHtml 関数

=== 所在

`src/utils/xss.js`

=== 責務

DOM への文字列反映前に HTML 特殊文字をエスケープし、XSS 攻撃を防ぐ。
クラスではなく単体の名前付き関数として定義する。

=== シグネチャ

[source,javascript]
----
export function escapeHtml(str)
----

[cols="1,3", options="header"]
|===
|引数
|説明

|`str`
|任意の値。非文字列の場合は空文字を返す。
|===

*変換表:*

[cols="1,1", options="header"]
|===
|入力文字
|変換後エンティティ

|`&`
|`&amp;`

|`<`
|`&lt;`

|`>`
|`&gt;`

|`"`
|`&quot;`

|`'`
|`&#039;`
|===

*実装方針:*
- 各文字を個別に `replace` でチェーン置換する
- `typeof str !== 'string'` の場合は即座に空文字を返す

== 2. validateRecord 関数

=== 所在

`src/utils/validator.js`

=== 責務

収支レコード1件の入力値を検証し、エラーコードの配列を返す。

=== シグネチャ

[source,javascript]
----
export function validateRecord({ date, type, category, amount })
----

[cols="1,1,3", options="header"]
|===
|引数フィールド
|型
|説明

|`date`
|`string`
|日付文字列。`YYYY-MM-DD` 形式であること

|`type`
|`string`
|種別。`RECORD_TYPES` 定数の値（`income` / `expense`）であること

|`category`
|`string`
|カテゴリ。`INCOME_CATEGORIES` または `EXPENSE_CATEGORIES` に含まれること

|`amount`
|`number\|string`
|金額。正の整数であること
|===

*検証ルール:*

[cols="1,2,2", options="header"]
|===
|フィールド
|判定条件
|エラーコード（ERROR_CODES）

|`date`
|`/^\d{4}-\d{2}-\d{2}$/` に一致すること
|`INVALID_DATE`

|`type`
|`Object.values(RECORD_TYPES)` に含まれること
|`INVALID_TYPE`

|`category`
|`INCOME_CATEGORIES + EXPENSE_CATEGORIES` に含まれること
|`MISSING_CATEGORY`

|`amount`
|正の整数（`num > 0 && Number.isInteger(num)`）であること
|`INVALID_AMOUNT`
|===

*戻り値:* エラーコード文字列の配列。全て正常な場合は空配列を返す。

== 3. validateBudget 関数

=== 所在

`src/utils/validator.js`

=== 責務

予算設定1件の入力値を検証し、エラーコードの配列を返す。

=== シグネチャ

[source,javascript]
----
export function validateBudget({ category, limit })
----

[cols="1,1,3", options="header"]
|===
|引数フィールド
|型
|説明

|`category`
|`string`
|カテゴリ。全カテゴリ（収入+支出）に含まれること

|`limit`
|`number\|string`
|予算上限金額。0以上の整数であること
|===

*検証ルール:*

[cols="1,2,2", options="header"]
|===
|フィールド
|判定条件
|エラーコード（ERROR_CODES）

|`category`
|`INCOME_CATEGORIES + EXPENSE_CATEGORIES` に含まれること
|`MISSING_CATEGORY`

|`limit`
|0以上の整数（`num >= 0 && Number.isInteger(num)`）であること
|`INVALID_AMOUNT`
|===

*戻り値:* エラーコード文字列の配列。全て正常な場合は空配列を返す。

*`validateRecord` との違い:* `limit` は 0 を許可する（「予算なし」を表すため）。

== 4. StorageManager クラス

=== 所在

`src/utils/storage.js`

=== 責務

localStorage への収支レコード・予算データの読み書きを一元管理する。

[cols="1,3", options="header"]
|===
|メソッド
|説明

|`getRecords()`
|`moneybook_records` キーから収支レコード配列を取得する。データが存在しない場合は空配列を返す

|`saveRecords(records)`
|収支レコード配列を `moneybook_records` キーに保存する

|`getBudgets()`
|`moneybook_budgets` キーから予算オブジェクトを取得する。データが存在しない場合は空オブジェクトを返す

|`saveBudgets(budgets)`
|予算オブジェクトを `moneybook_budgets` キーに保存する
|===

---

== 5. 使用上の共通ルール

. 画面表示する全文字列は `escapeHtml()` を通してから DOM に反映する
. フォーム送信時に `validateRecord()` / `validateBudget()` を呼び出し、エラーがある場合は処理を中止する
. localStorage の読み書きは必ず `StorageManager` クラス経由で行い、他所で直接操作しない
