== テーブル基本情報

[#テーブル基本情報, title="テーブル基本情報", cols="2,5", options="header", align="center"]
|===
^.^|項目 ^.^|内容

<.^|テーブルID（物理名） <.^|moneybook_records
<.^|テーブル名（論理名） <.^|収支レコードテーブル
<.^|保存先 <.^|localStorage
<.^|保存形式 <.^|JSON 配列（Array<Object>）
<.^|保存キー <.^|`moneybook_records`
<.^|用途 <.^|家計簿の収入・支出レコードを一元管理する。月次フィルタリング・集計・CSV エクスポートの元データとして使用する
<.^|主な操作クラス <.^|`BudgetService`（追加・更新・削除・集計）、`StorageManager`（読み書き）
|===

== カラム定義

[#カラム定義, title="カラム定義", cols="1,3,3,2,1,1,3,4", options="header", align="center"]
|===
^.^|No. ^.^|カラム名（論理名） ^.^|カラム名（物理名） ^.^|型 ^.^|必須 ^.^|主キー ^.^|デフォルト値 ^.^|備考

^.^|{counter:カラム} <.^|レコードID <.^|id <.^|string <.^|○ <.^|PK <.^|- <.^|`$\{Date.now()}-$\{Math.random().toString(36).slice(2,8)}` 形式で自動採番。ミリ秒衝突を乱数6桁で回避する
^.^|{counter:カラム} <.^|日付 <.^|date <.^|string <.^|○ <.^|- <.^|- <.^|YYYY-MM-DD 形式（例: `2026-04-06`）。月次フィルタリングは `date.startsWith(YYYY-MM)` で実装
^.^|{counter:カラム} <.^|種別 <.^|type <.^|string <.^|○ <.^|- <.^|- <.^|`'income'`（収入）または `'expense'`（支出）のいずれか。`RECORD_TYPES` 定数で管理
^.^|{counter:カラム} <.^|カテゴリ <.^|category <.^|string <.^|○ <.^|- <.^|- <.^|`INCOME_CATEGORIES` または `EXPENSE_CATEGORIES` に含まれる値のみ許可。バリデーション: `allCategories.includes(category)`
^.^|{counter:カラム} <.^|金額 <.^|amount <.^|number <.^|○ <.^|- <.^|- <.^|1以上の正の整数。`Number(data.amount)` で数値変換して保存。バリデーション: `num > 0 && Number.isInteger(num)`
^.^|{counter:カラム} <.^|メモ <.^|memo <.^|string <.^|- <.^|- <.^|`''`（空文字） <.^|任意入力。未入力時は空文字で保存する
^.^|{counter:カラム} <.^|作成日時 <.^|createdAt <.^|string <.^|○ <.^|- <.^|- <.^|ISO 8601 形式（例: `2026-04-06T12:34:56.789Z`）。`new Date().toISOString()` で自動生成。更新時は変更しない
|===

== バリデーション制約

[#バリデーション制約, title="バリデーション制約", cols="2,3,3,4", options="header", align="center"]
|===
^.^|カラム名（物理名） ^.^|制約種別 ^.^|制約内容 ^.^|エラーコード

<.^|date <.^|形式チェック <.^|`/^\d{4}-\d{2}-\d{2}$/` にマッチすること <.^|`ERROR_CODES.INVALID_DATE`
<.^|date <.^|必須チェック <.^|空文字・null・undefined を不可とする <.^|`ERROR_CODES.INVALID_DATE`
<.^|type <.^|列挙値チェック <.^|`'income'` または `'expense'` のみ許可 <.^|`ERROR_CODES.INVALID_TYPE`
<.^|category <.^|列挙値チェック <.^|`INCOME_CATEGORIES` + `EXPENSE_CATEGORIES` の結合リストに含まれること <.^|`ERROR_CODES.MISSING_CATEGORY`
<.^|amount <.^|数値チェック <.^|数値変換後 NaN でなく、0より大きく、整数であること <.^|`ERROR_CODES.INVALID_AMOUNT`
|===
